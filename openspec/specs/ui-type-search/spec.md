## ADDED Requirements

### Requirement: 可搜索的 Proto 类型选择器
系统 SHALL 提供可搜索的 Proto 类型下拉选择器，支持模糊匹配。

#### Scenario: 展开下拉列表
- **WHEN** 用户点击类型选择器
- **THEN** 系统显示所有可用类型的下拉列表

#### Scenario: 输入搜索过滤
- **WHEN** 用户在选择器中输入搜索文本
- **THEN** 系统通过后端 API 搜索，只返回类型名包含搜索文本的项（忽略大小写）

#### Scenario: 搜索结果分页
- **WHEN** 搜索返回的结果数量超过每页显示数量
- **THEN** 系统显示分页控件，用户可翻页查看搜索结果

#### Scenario: 选择类型
- **WHEN** 用户从列表中选择一个类型
- **THEN** 系统填充选择器并触发相关的解码操作

#### Scenario: 键盘导航
- **WHEN** 用户使用键盘上下键导航列表
- **THEN** 系统高亮当前选项，按 Enter 选中

### Requirement: 已加载类型列表固定显示
系统 SHALL 以固定高度的列表形式展示已加载的 Proto 类型。

#### Scenario: 显示类型列表
- **WHEN** 用户加载了多个 proto 文件后
- **THEN** 系统在固定高度的滚动区域内显示类型列表，每项显示类型名和删除按钮

#### Scenario: 分页浏览
- **WHEN** 已加载类型数量超过每页显示数量（默认 10 条）
- **THEN** 系统显示分页控件，用户可翻页查看
